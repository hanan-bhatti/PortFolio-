# SECURITY.md - Security Policy & Vulnerability Disclosures

We take security seriously. If you discover a vulnerability in this repository, please report it following the guidelines below.

---

## 🔒 Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues. Instead, report them privately:

1. **Email Contact**: Send an email to the administrator (use the public contact email set in the application layout or your configurations).
2. **Details to Include**:
   * Description of the vulnerability and the potential impact.
   * Step-by-step instructions to reproduce the issue (including proof of concept scripts, HTTP requests, or screenshots).
   * Details about your testing environment (browser, OS version).

---

## ⏱️ Expected Response Times

* **Acknowledgment**: You can expect an email acknowledgment within **48 hours** of submission.
* **Resolution / Status Update**: We aim to provide status updates and details regarding resolution paths within **7 days** of initial verification.
* **Coordinated Disclosure**: We request that you follow coordinated disclosure practices and give us reasonable time to patch the issue before making it public.

---

## 🎯 Scope

This policy applies exclusively to the code contained in this repository and does not cover external third-party integrations (such as Neon PostgreSQL servers, Resend APIs, or UploadThing networks). Vulnerabilities in those services should be reported directly to their respective security teams.
